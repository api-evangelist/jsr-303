Abstract The Jakarta EE ecosystem offers specifications for building database backend applications using relational and/or NoSQL databases. The Jakarta Persistence specification, with a rich history dating back almost 20 years, has been a staple in the Java community for writing relational database applications. The Jakarta NoSQL specification, with a short history dating back to 2020, provides comprehensive support for all four types of NoSQL databases.
